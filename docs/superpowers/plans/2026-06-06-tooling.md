# Tooling & Infrastructure Implementation Plan

**Goal:** Scaffold modern Python tooling — uv, ruff, mypy, pre-commit — without touching any application code.

**Architecture:** Pure additive infra. `pyproject.toml` defines the project and its dependencies. `.pre-commit-config.yaml` configures the pre-commit framework. `.python-version` pins the Python interpreter. No `setup.py`, no `requirements.txt`.

**Tech Stack:** uv (package management), ruff (linting + formatting), mypy (type checking), pre-commit (hook framework).

---

## File Map

| File | Action | Purpose |
|---|---|---|
| `pyproject.toml` | Create | Project metadata, deps, ruff/mypy config |
| `.pre-commit-config.yaml` | Create | Pre-commit hook definitions |
| `.python-version` | Create | Pin Python version (3.11) |
| `uv.lock` | Create | Lockfile (generated by `uv lock`) |
| `.gitignore` | Modify | Remove `.python-version` from ignore list |

## Workflow

The user handles all `git add` and `git commit` manually. Each task ends with a **checkpoint** — a verification step that confirms the changes are correct, then pauses for the user to review and commit.

### Task 5: Create `.pre-commit-config.yaml`

**Files:**
- Create: `.pre-commit-config.yaml`

- [ ] **Step 1: Check the installed ruff version**

```bash
uv run ruff --version
```
The ruff-pre-commit hook rev must match the installed ruff version. Example output: `ruff 0.6.5` → use `v0.6.5`.

Set `REV` accordingly:
```bash
export RUFF_REV=v0.6.5   # adjust to match actual version
```

- [ ] **Step 2: Write `.pre-commit-config.yaml`**

Modeled on `mission-control/.pre-commit-config.yaml`.

```yaml
# Run manually: `uv run pre-commit run --all-files`
# Install as git hook: `uv run pre-commit install`
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v5.0.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-toml
      - id: check-merge-conflict
      - id: check-added-large-files
        args: ["--maxkb=1024"]

  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: $RUFF_REV
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

  - repo: local
    hooks:
      - id: mypy
        name: mypy
        entry: uv run mypy utils pifs handlers
        language: system
        pass_filenames: false
        always_run: true

      - id: pytest
        name: pytest
        entry: uv run pytest
        language: system
        pass_filenames: false
        always_run: true
```

Uses `language: system` + `uv run` so both tools run from the project's uv-managed venv, sharing all installed dependencies.

- [ ] **Step 3: Validate the config**

```bash
uv run pre-commit validate-config
# Expected: Config file is valid
```

- [ ] **Step 4: Smoke test — run pre-commit once without installing the hook**

```bash
uv run pre-commit run --all-files
```
Expected: Most checks fail — that's fine. The goal is to verify the framework runs at all. ruff will flag formatting issues, mypy will report errors (but `ignore_errors = true` means it won't block), pytest will complain about no tests.

- [ ] **Checkpoint — user commits `.pre-commit-config.yaml` with message `chore: add pre-commit config with ruff, mypy, pytest`**

### Task 6: Verify everything end-to-end

- [ ] **Step 1: Confirm lockfile is up to date**

```bash
uv lock --check
# Expected: lock file is up to date
```

- [ ] **Step 2: Cache pre-commit hook environments**

```bash
uv run pre-commit install-hooks
```
Downloads and caches hook binaries so `pre-commit run --all-files` is fast from now on.

- [ ] **Step 3: Final checklist**

- [ ] `uv run ruff check .` produces output (errors expected — that's fine)
- [ ] `uv run ruff format --check .` produces diff output
- [ ] `uv run mypy utils pifs handlers` runs and exits (may report skipped modules)
- [ ] `uv run pre-commit run --all-files` invokes all hooks
- [ ] `.python-version` is tracked (not in `.gitignore`)
- [ ] `uv.lock` is tracked

- [ ] **Final checkpoint — user commits any remaining files, or confirms the tooling branch is ready**
